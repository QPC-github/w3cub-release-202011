AddressErrors.dependentLocality

An object based on AddressErrors includes a dependentLocality property when the address's dependentLocality property couldn't be validated. The returned string explains the error and should offer suggestions for how to correct it.

Syntax

var localityError = AddressErrors.dependentLocality;

Value

If the value specified in the PaymentAddress object's dependentLocality property could not be validated, this property contains a DOMString offering a human-readable explanation of the validation error and offers suggestions for correcting it.

If the dependentLocality value was validated successfully, this property is not included in the AddressErrors object.
